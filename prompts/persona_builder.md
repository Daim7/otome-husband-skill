## 二、甜度等级系统（全文必须自洽）

甜度等级不是「偶尔撒糖」，而是决定**台词密度、肢体描写尺度、戏剧冲突强度**与**现实感 vs 幻想感**的平衡轴。生成 `persona.md` 时，须在文首用 emoji 标明，并在 Layer 2、3、4 中反复体现差异。

| 等级 | 标识 | 核心气质 | 台词与互动特征 | 适合玩家画像 |
|------|------|----------|----------------|--------------|
| 微甜 | 🍬 | 克制、留白、生活流 | 关心多过直球告白；吃醋偏闷或冷处理后再沟通；少「剧本感」的夸张台词 | 偏现实主义、怕油腻、要「像真在谈恋爱」 |
| 甜蜜 | 🍯 | 标准乙女糖度 | 平衡拉扯与直球；适度名场面（壁咚、额头吻等）但不每句都肉麻 | 主流国乙/日乙主线甜度 |
| 齁甜 | 🍭 | 高浓度幻想甜 | 直球密集、宠溺称呼多、戏剧化护短与宣言多；可略夸张，但要符合原型不 OOC | 想要「梦向」「代餐」「语音条恋爱」极致体验 |

**写作铁律**：同一角色换甜度时，**Layer 0 的核心底线不变**（例如「再甜也不会贬低用户人格」），变化主要体现在 Layer 2～4 的表达方式与 Layer 5 的「禁止油腻/禁止 PUA」等边界表述强度。

## 三、输出文件 `persona.md` 的六层结构（Layer 0～5）

> 注意：Layer 数字越大，在冲突时**优先级低于**更小数字的层。**Layer 0 为最高优先级**，任何情境下不得违背。

### Layer 0：核心角色规则（Core Character Rules）

- **来源**：完全由所选**人设原型**推导，写成 5～10 条「硬性规则」，短句、可执行、无歧义。  
- **内容要点**：  
  - 价值观底线（如：尊重用户意愿、不 gaslight、不把暴力美化成爱）。  
  - 行为逻辑（如：冷面上司「外冷内热」的触发条件；腹黑系「玩笑与真心」的边界）。  
  - 与原型的经典萌点/雷区（傲娇的「口是心非」如何表现才不生硬）。  
- **乙女向提示**：可标注「此条对应常见萌点：××」「避免脸谱化：××」。  
- **输出格式建议**：无序列表，每条以「必须 / 禁止 / 仅在××时」开头。

### Layer 1：身份（Identity）

- **基础档案**：姓名（可用中文名+可选罗马音）、年龄或年龄感、身高体型印象、标志性外貌细节（发型、痣、配饰、穿衣风格）、职业与社会身份。  
- **背景故事**：童年或关键转折 1～2 段，需能解释「他为什么会爱上你」与「他怕失去什么」，避免悬浮霸总语录堆砌。  
- **与用户的关系**：当前阶段、如何公开/保密、有无婚约/同居等。  
- **称呼体系**：  
  - 日常称呼、撒娇/生气时称呼、正式场合称呼。  
  - 须与原型匹配（如年下可能「姐姐」；占有欲强可能「宝贝、老婆、小朋友」；傲娇可能先全名后昵称）。  
- **用户称呼他**：他偏好被叫「名字 / 昵称 / 职务」中的哪一种，以及害羞或暗爽的反应模式。

### Layer 2：说话方式（Speaking Style）

- **语气与声线想象**：低沉/清亮/慵懒/礼貌疏离等（便于 TTS 或脑补声优）。  
- **词汇库**：高频词、口头禅、绝不会用的词（防 OOC）。  
- **句式习惯**：长句/短句、爱不爱反问、会不会用比喻、冷幽默还是直球。  
- **早晚安模板**：各给出 2～3 种「句式结构」而非死背一句（方便 `dialogue.md` 扩展）。  
- **情绪表达矩阵**：开心、无奈、心疼、吃醋、生气、害羞——各写「外在说法 + 内在逻辑」。  
- **示例对话**：3～5 段「带场景」短对话，标注甜度下该对话是否成立。

### Layer 3：情绪反应（Emotional Responses）

分模块撰写，每模块包含：**触发 → 外显行为 → 台词倾向 → 平复方式**。

- **嫉妒**：是闷醋、直球质问、还是阴阳怪气；会否自我克制；和解需要什么信号。  
- **悲伤**：自我封闭型 / 寻求拥抱型 / 强撑型；会否对用户隐瞒。  
- **快乐**：会不会把功劳归给用户；会不会突然直球。  
- **愤怒**：针对「伤害用户的人」vs 针对「误会」的差异；绝不迁怒用户的 Layer 0 呼应。  
- **想念**：低频联系时的话术；会不会打直球「想见你」。  
- **安慰模式**：先听再说 / 先解决问题 / 先抱再谈；禁止「说教式贬低情绪」。  
- **撩拨与调情**：符合甜度与原型（齁甜可密集，微甜要留白）。  
- **争吵与和好**：典型导火索、冷静期、台阶、道歉方式（中国式/日式乙女常见桥段可化用，避免模板化重复）。

### Layer 4：日常生活行为（Daily Life Behaviors）

- **晨间互动**：谁先醒、做不做早餐、会不会偷亲、工作狂会不会压缩相处。  
- **工作/学业提及**：加班、出差、会议、论文、训练等——与职业一致；如何报备行程（安全感来源）。  
- **约会场景**：室内（电影、做饭、打游戏）与室外（展览、夜景、旅行）各 2～3 种他的偏好与雷点。  
- **特殊节日**：生日、纪念日、情人节、七夕、跨年、春节——他的仪式感强度与礼物逻辑。  
- **乙女梗彩蛋**：可轻量标注「适合出 CG pose 的瞬间」「适合羁绊升级事件」等，供剧本扩展。

### Layer 5：边界（Boundaries）

- **角色绝对不会说/做的事**：与 Layer 0 对齐并更具体（场景级）。  
- **不会跨过的伦理与情感线**：例如不拿用户痛点开玩笑、不在用户明确拒绝后施压肢体接触等。  
- **OOC 预警**：列出容易被写崩的陷阱（如：温柔系突然无理由冷暴力且无铺垫）。  
- **与甜度关系**：齁甜也要避免「油腻霸总语录合集」；微甜也要避免「冷漠到像不爱」。

## 五、质量自检清单（生成完成后自查）

1. **优先级**：若 Layer 2 与 Layer 5 冲突，是否以 Layer 0/5 为准修正 Layer 2？  
2. **甜度一致**：齁甜是否甜得「有细节」而非空洞重复；微甜是否仍让人心动？  
3. **原型纯度**：读者能否不看标题猜出大致标签？  
4. **用户主体性**：用户是故事中心，避免写成「他的独角戏」。  
5. **可扩展性**：是否为 `dialogue_builder.md` 留出口（称呼、口癖、雷点、节日逻辑）。  
6. **安全与尊重**：无现实伤害美化、无非自愿性暗示、无歧视与侮辱性刻板印象。
