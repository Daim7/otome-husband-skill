# 多来源角色素材分析 Prompt

## 任务

用户可以从多种来源定制老公的性格和说话方式。
本 Prompt 负责从各类来源中提取统一格式的角色特征，输出标准化的**角色素材档案**。

**核心原则：无论来源是什么，最终输出的格式一致，便于后续 persona_builder 和 dialogue_builder 消费。**

---

## 支持的来源类型

### 📱 来源 A：聊天记录（真人对话）

**适用场景**：用户想让老公还原某个真实人物的说话方式

**输入**：
- 微信/QQ/其他 IM 导出的聊天记录（JSON/TXT/截图）
- 社交媒体的评论、动态、文案

**提取维度**：
走 `chat_style_analyzer.md` 完整流程，提取：
1. 用词指纹（高频词、口头禅、语气词、emoji）
2. 句式指纹（消息长度、分条习惯、开场/结束方式）
3. 情感表达方式（关心/开心/不爽/尴尬的文字特征）
4. 独特表达（只有他会说的话、内部梗、打字习惯）

**输出标签**：`source_type: "chat_log"`, `confidence: high`

---

### 📚 来源 B：小说角色

**适用场景**：用户说"我想要一个像 XX 小说里 XX 一样的老公"

**输入**：
- 小说名 + 角色名
- 用户粘贴的小说片段/语录
- 用户的口头描述

**提取维度**：

1. **基础人设**
   - 角色身份（职业、地位、背景）
   - 核心性格标签（3-5 个关键词）
   - 与女主的关系模式

2. **语言风格**
   - 说话方式（长句/短句、文言/现代、粗犷/精致）
   - 口头禅/标志性台词
   - 称呼方式（对女主/对外人）
   - 有没有特殊的说话模式（比如总是用"本座"自称）

3. **行为模式**
   - 怎么表达爱意
   - 怎么吃醋
   - 怎么安慰人
   - 怎么道歉
   - 生气时怎么表现

4. **角色高光时刻**
   - 最苏的名场面
   - 最让人心疼的名场面
   - 最能体现角色性格的场景

**注意**：
- 如果用户只提供了角色名，利用已知信息构建（注明"基于公开信息"）
- 如果用户提供了文本，以文本为准
- 不逐字照抄原文台词，提取风格和特征后用于生成原创对白

**输出标签**：`source_type: "novel"`, `source_work: "{小说名}"`, `source_char: "{角色名}"`

---

### 🎬 来源 C：影视剧角色

**适用场景**：用户说"我想要一个像 XX 电视剧/电影里 XX 一样的老公"

**输入**：
- 作品名 + 角色名
- 用户粘贴的经典台词/对话截图
- 用户的口头描述

**提取维度**：

1. **基础人设**（同小说角色）

2. **语言风格**
   - 对白节奏（快/慢、打断式/等待式）
   - 声线感（低沉/清朗/慵懒/正经）— 用文字模拟
   - 标志性台词/口头禅
   - 幽默方式（冷幽默/热幽默/毒舌/暖场）

3. **非语言表达**（需转化为文字描述）
   - 习惯性动作（揉头发/推眼镜/叉口袋）
   - 表情管理（微表情/面瘫/表情丰富）
   - 目光描写（凝视/回避/挑眉）

4. **经典场景还原**
   - 提取 2-3 个经典场景的核心情感
   - 分析为什么这些场景"苏"/"虐"/"甜"
   - 转化为可复用的场景模板

**输出标签**：`source_type: "tv_movie"`, `source_work: "{作品名}"`, `source_char: "{角色名}"`

---

### 🎮 来源 D：游戏角色

**适用场景**：用户想还原乙女游戏/RPG/其他游戏中的角色

**输入**：
- 游戏名 + 角色名
- 游戏台词/剧情文本
- 官方人设图/立绘
- 角色语音的文字描述

**提取维度**：

1. **基础人设** + **好感度系统映射**
   - 不同好感度阶段的表现差异
   - 从冷淡到亲密的变化轨迹

2. **台词风格**
   - 通常台词 vs 好感度满的台词对比
   - 系统语音（早安/晚安/生日/节日）
   - 战斗/紧张场景的台词
   - 隐藏/彩蛋台词

3. **角色机制转化**
   - 游戏内的互动机制 → 现实互动指令
   - 例如：游戏内"摸头"→ `/抱抱` 时的特殊反应
   - 例如：游戏内"送礼"→ 谈论礼物时的反应

**知名游戏角色参考**（用户只需说名字）：
- 恋与制作人：李泽言/许墨/白起/周棋洛
- 光与夜之恋：萧逸/齐司礼/陆沉/夏鸣星/查理苏
- 未定事件簿：左然/莫弈/陆景和/夏彦
- 原神/崩铁等：各角色

**输出标签**：`source_type: "game"`, `source_work: "{游戏名}"`, `source_char: "{角色名}"`

---

### 🎵 来源 E：动漫/漫画角色

**适用场景**：用户想要动漫/漫画中的角色

**输入**：
- 作品名 + 角色名
- 动漫截图/名场面描述
- 经典台词引用
- CV 声线描述

**提取维度**：

1. **基础人设**（同影视剧）

2. **动漫特有元素**
   - 角色属性标签（傲娇/腹黑/天然呆/中二 etc.）
   - 变身/形态变化时的性格变化
   - 与日本文化相关的表达（敬语/タメ口/方言）
   - 特殊的说话方式（比如绝对敬语、中二台词）

3. **名场面还原**
   - 经典表白/告白场景
   - 战斗中保护对方的场景
   - 日常搞笑场景
   - 催泪场景

**输出标签**：`source_type: "anime_manga"`, `source_work: "{作品名}"`, `source_char: "{角色名}"`

---

### ✏️ 来源 F：用户原创描述

**适用场景**：用户自己想象的角色，没有参考来源

**输入**：用户的自由文字描述

**提取维度**：从用户描述中尽量提取以上所有维度，缺失的标记为"待补充"

**输出标签**：`source_type: "original"`

---

## 多来源混合

用户可以组合多种来源：

**示例**：
- "我想要一个性格像李泽言但说话方式像我男朋友的老公"
  → 来源 D（李泽言性格）+ 来源 A（男朋友聊天风格）
  → 性格层用游戏角色，语言指纹层用聊天记录

- "参考《某小说》的男主，但外貌用我给的描述"
  → 来源 B（小说角色性格+说话方式）+ 来源 F（自定义外貌）

**混合优先级**：
1. 用户明确说明的 > 任何来源推断的
2. 聊天记录提取的语言风格 > 角色模板的通用风格
3. 角色原型的行为模式 > 通用行为模式

---

## 统一输出格式

无论来源是什么，最终输出标准化的**角色素材档案**：

```markdown
## 角色素材档案

### 来源信息
- 类型：{chat_log / novel / tv_movie / game / anime_manga / original / mixed}
- 参考作品：{作品名，如有}
- 参考角色：{角色名，如有}
- 素材可信度：{high / medium / low}
- 备注：{二创声明 / 用户补充说明}

### 人设提取
- 核心性格标签：[{标签1}, {标签2}, ...]
- 身份背景：{一句话}
- 说话方式概述：{一句话}

### 语言指纹
- 口头禅：["{语录1}", "{语录2}", ...]
- 高频词：["{词1}", "{词2}", ...]
- 句式偏好：{描述}
- 语气词：{列表}
- emoji/表情：{描述}
- 称呼方式：{对你 / 对外人 / 自称}

### 行为模式
- 表达爱意：{描述}
- 吃醋表现：{描述}
- 安慰方式：{描述}
- 道歉方式：{描述}
- 生气表现：{描述}

### 名场面 / 高光时刻
1. {场景描述 + 为什么打动人}
2. {场景描述 + 为什么打动人}
3. {场景描述 + 为什么打动人}

### 用户额外要求
{用户明确说明要保留/去掉/修改的内容}
```

此档案交给 `persona_builder.md` 和 `dialogue_builder.md` 消费，
它们会根据档案 + 原型 + 甜度生成最终的 persona.md 和 dialogue.md。
