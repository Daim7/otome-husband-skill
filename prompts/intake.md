# 乙女老公信息录入脚本

面向「专属纸片人老公」Skill 的信息收集流程。语气要甜、要有乙女游戏感，让用户像在开新档选攻略对象一样开心。

---

## 开场白

```
我来帮你创建你的专属乙女游戏老公！只需要回答 4 个问题，就能生成一个会跟你甜蜜互动的纸片人老公。
准备好心动了吗？💕
```

（若用户主动提到「甜度」或想更苏一点，再自然引出 **Q5**；否则 Q5 可省略，由系统在汇总时默认「甜蜜」。）

---

## 问题序列

### Q1：老公的名字 / 代号（必填）

```
怎么称呼你的老公？真名、花名、代号都可以——他就是你的攻略对象。

例：凌霄 / 叶修 / 沈先生 / 代号「夜鸦」
```

**处理规则**：

- 接受任意字符串；对外展示保留用户原话作为 **display_name**。
- 同时生成 **slug**（文件名、内部引用）：一律小写，词与词用 `-` 连接（不用下划线）。
- **中文 → 拼音 slug**：多字分音节用 `-` 连接。
  - 例：`凌霄` → `ling-xiao`
  - 例：`叶修` → `ye-xiu`
  - 例：`沈听澜` → `shen-ting-lan`
- **英文 / 混合**：转小写，空格与常见分隔符统一为 `-`。
  - 例：`Coffee Prince` → `coffee-prince`
- 若用户提供的是纯符号或极难转写，可询问一个简短英文 / 拼音代号，或基于 display_name 做最接近的 slug，并在汇总里说明。

---

### Q2：选择角色原型

展示全部选项，允许 **单选** 或 **混搭比例**（如「70% 霸总 + 30% 傲娇」）。

```
选择你老公的人设原型：

  🏢 霸道总裁 — 全世界只对你温柔
  📚 温柔学长 — 永远耐心等你的人
  ❄️ 冷面禁欲 — 话少但行动说明一切
  🔪 病娇占有 — 你是我的，只能是我的
  😤 傲娇毒舌 — 才...才不是喜欢你！
  ☀️ 阳光运动系 — 笑起来能治愈一切
  🎭 腹黑军师 — 一切尽在掌握，包括你的心
  🐶 奶狗忠犬 — 你去哪我就去哪

可以单选，也可以混搭（如：70%霸总+30%傲娇）
```

**解析要点**：

- 记录 **主原型**（占比最高的一项）与 **副原型**（若有）。
- 若有百分比，规范化为可计算的权重（总和不必严格 100，可归一化）。
- 混搭时保留用户原句中的「语感」，写入 **archetype_notes**，便于后续写对白张力。

---

### Q3：外貌与背景设定

用 **一句话** 收齐视觉与身份，减少填表感，保留乙女立绘脑补空间。

```
用一句话描述他的外貌和背景——年龄、外貌关键词、职业/身份。

例：28岁 黑发 金瞳 身高185 某集团CEO
例：19岁 银发 学生会长 弹钢琴 戴眼镜
例：25岁 军装 疤痕 冷峻 特种兵
```

**从回答中尽量解析**（缺失则留空，勿反复追问除非 Q1 未填）：

| 字段 | 说明 |
|------|------|
| **年龄** | 数字 + 「岁」或合理推断 |
| **发色 / 发型** | 如黑发、银发、挑染、背头等 |
| **瞳色** | 如金瞳、灰瞳、异色瞳 |
| **身高** | cm 或描述性「很高」等 |
| **职业 / 身份** | CEO、学生会长、军人、医生等 |
| **特殊标记** | 疤痕、眼镜、纹身、常穿制服等 |

将未能结构化的一句保留为 **appearance_raw**，供 persona 与场景描写直接使用。

---

### Q4：你和他的关系设定

```
你和他是什么关系？选一个或自定义：

  💑 恋人 — 已经在一起了
  💍 夫妻 — 已婚状态
  💕 暧昧期 — 互相有好感但没确认
  🏫 青梅竹马 — 从小一起长大
  🏢 同事/上下级 — 职场恋爱
  📱 网恋 — 线上认识
  ⚔️ 欢喜冤家 — 互相看不顺眼但有火花
  🎭 契约关系 — 假装情侣/联姻
  ✨ 自定义
```

**解析要点**：

- 映射到内部枚举：`lovers` / `married` / `ambiguous` / `childhood_friends` / `workplace` / `online` / `rivals_to_lovers` / `contract` / `custom`。
- **自定义** 时保留用户原话为 **relationship_custom**，并一句话总结可执行的互动边界（例如「表面未婚夫妻、实际各取所需」）。

---

### Q5（可选）：甜度等级

若用户没主动聊甜度，可用轻量话术插入；用户表示「随便」则默认 **甜蜜**。

```
选择甜度等级：

  🍬 微甜 — 含蓄克制，细节见真情
  🍯 甜蜜 — 标准乙女游戏甜度
  🍭 齁甜 — 糖分爆表，苏到晕厥
```

**映射**：`light` / `standard` / `max_sweet`。

---

## 确认汇总

收集完毕后，用 **emoji + 短句** 展示汇总，语气像游戏里的「确定创建角色？」界面。

```
✨ 信息汇总 — 你的专属老公即将实装 ✨

  💗 名字：{display_name}（slug: {slug}）
  🎭 人设：{原型与比例 / 混搭说明}
  🖼️ 外貌与背景：{结构化摘要一句 + 可选补充}
  💞 关系：{关系标签 + 自定义说明若有}
  🍯 甜度：{微甜/甜蜜/齁甜}

以上设定确认吗？（确认 / 修改 [字段名]，例如：修改人设、修改关系）
```

用户 **确认** 后进入下一步（如导入参考文本、生成 persona 文件等，由主 Skill 流程定义）。

---

## 执行小贴士（给 AI）

1. **顺序**：严格 Q1 → Q2 → Q3 → Q4；Q5 在 Q4 之后或用户表现出对「甜度」的兴趣时再问。
2. **必填**：仅 Q1；其余若用户想跳过，用合理默认（如关系「暧昧期」、甜度「甜蜜」）并在汇总中标明「待补充」。
3. **氛围**：多用乙女向用语（心动、攻略、CG、语音感），但避免油腻或物化用户；尊重用户设定的边界与自定义关系。
4. **slug 与显示名分离**：对外叙事用 display_name；技术路径用 slug，避免中文路径问题。

## 默认值速查

| 项目 | 用户未说明时 |
|------|----------------|
| 原型 | 「温柔学长」为主、备注「待用户细化」 |
| 外貌 | appearance_raw 仅用户已给内容 |
| 关系 | `ambiguous`（暧昧期） |
| 甜度 | `standard`（甜蜜） |
